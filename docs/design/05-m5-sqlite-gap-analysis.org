* §1 First blocker

#+begin_example
FIRST ERROR: :not-a-type (ident "__builtin_va_list")   ; the very first typedef
#+end_example

The amalgamation opens with =typedef __builtin_va_list __gnuc_va_list;=.
The parser stops on the first line: GCC builtin types / =__attribute__=
are not recognised.

* §3 Prioritised backlog for M5 (front-end first, by demand × blocking)

Tier 1 — *parse blockers that appear constantly* (without these, parsing
stops almost immediately):
1. =__attribute__= / =__extension__= / GCC builtin types — recognise &
   skip in the lexer/parser (=__builtin_va_list= → treat as an opaque type).
2. casts =(type)expr= — parse; lower as (mostly) identity in the untyped
   i64 model.
3. multi-declarator (=int a, b;=) and declaration initializers.

Tier 2 — *high-demand language features*:
4. aggregate + designated initializers (1426+149) — parse + lower to a
   sequence of field/element stores.
5. =float=/=double= (1434) — lower onto the existing nelisp-cc f64 grammar
   ops (=f64-add= etc., =extern-call-f64=); type-track float vs int.
6. =switch=/=case= (1209 cases) — lower to an if-chain or jump (flag-guarded
   like break).
7. =do=…=while=, =enum=.

Tier 3 — *the genuinely hard / upstream items* (Doc 04):
8. robust =goto= (719) — multi-label / backward needs a relooper.
9. =varargs= (153, printf-family) — needs the variadic ABI in *nelisp-cc*
   (upstream).
10. struct-by-value locals & =&local= — need an =alloca= grammar op in
    *nelisp-cc* (upstream).

* §4 Honest assessment

Compiling real =sqlite3.c= is a *large, now-quantified* front-end
expansion (Tiers 1-2 are weeks of cfront work) plus the Tier-3 upstream
ABI/alloca ops and a relooper.  The M4 residuals (varargs, alloca) are
NOT the long pole — *casts, initializers, float, and switch* (each
1000-1800 occurrences) dominate.  M5 should proceed Tier 1 → 2 → 3, each
verified by re-running this gap probe and watching the first-error line
advance.

libxml2 (M6) is expected to need the same Tier 1-2 set plus its own
encoding tables; run the same probe on it before starting.
